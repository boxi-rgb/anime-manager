# Workflow - anime-manager

## TDD Policy
- **柔軟 (Flexible)**: 複雑なロジックのみテストを実行する。基本は実装スピードを優先。

## Commit Strategy
- **git-committer**: `feat:`, `fix:` 等のプレフィックスを用いた Conventional Commits を採用。

## Code Review
- **AI Self-Review**: エージェントによるセルフレビューを基本とする。

## Verification
- **Track Completion Check**: 全ての実装が終了した段階で、まとめて手動検証を行う。
